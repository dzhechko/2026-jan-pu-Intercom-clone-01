# Пользовательские сценарии

## 1. Авторизация администратора

```
Администратор                    Система
     |                              |
     |  1. Открыть /login           |
     |----------------------------->|
     |                              |
     |  2. Ввести email + пароль    |
     |----------------------------->|
     |                              |
     |     3. POST /api/v1/auth/login
     |                              |
     |     4. Проверка учетных данных
     |                              |--- Верификация email
     |                              |--- Верификация пароля (bcrypt / dev)
     |                              |
     |  5. JWT-токен (bearer)       |
     |<-----------------------------|
     |                              |
     |  6. Сохранение токена        |
     |     в localStorage           |
     |                              |
     |  7. Редирект на /dashboard   |
     |<-----------------------------|
```

### Ключевые детали

- Email по умолчанию: `admin@cloud.ru`
- Dev-пароль: `admin123admin` (работает только при отсутствии `ADMIN_PASSWORD_HASH`)
- Токен содержит: email, роль (admin), tenant_id
- Срок действия: 60 минут (настраивается через `JWT_EXPIRE_MINUTES`)
- При истечении токена -- повторный вход

- Стоимость управляемого Kubernetes (control plane).
- Сравнение: Cloud.ru vs on-premise vs AWS/GCP (если применимо).
- Помесячная и годовая разбивка.

- Исчерпание максимального числа шагов диалога (`max_turns`).
- Специфические темы (юридические, финансовые гарантии).

### Критерии квалификации

| Уровень | Критерии |
|---------|----------|
| cold | Задан общий вопрос, контакт не предоставлен |
| warm | Задан конкретный технический вопрос, частичный контакт |
| qualified | Указан бюджет, сроки, полный контакт |
| hot | Запрошена встреча с менеджером, готов к сделке |

### Ежедневный чеклист администратора

1. Проверить метрики на дашборде -- нет ли аномального роста эскалаций.
2. Просмотреть список эскалированных разговоров -- все ли обработаны SA.
3. Проверить "горячие" лиды -- переданы ли в отдел продаж.
4. На странице ROI -- убедиться, что конверсия не падает.
5. Проверить health-эндпоинты -- все сервисы работают.
