# Руководство администратора

## Вход в систему

### Учетные данные для разработки

| Поле | Значение |
|------|----------|
| Email | admin@cloud.ru |
| Пароль | admin123admin |
| URL | http://localhost:3000 |

В продакшн-окружении используется bcrypt-хеш пароля через переменную `ADMIN_PASSWORD_HASH`. Dev-пароль работает только при отсутствии этой переменной.

### Процесс авторизации

1. Откройте панель администратора в браузере.
2. Введите email и пароль.
3. Система выдаст JWT-токен со сроком действия 60 минут.
4. Токен автоматически используется для всех последующих запросов к API.

## Панель управления (Dashboard)

### Метрики

Главная страница отображает ключевые показатели за выбранный период (сегодня, 7 дней, 30 дней):

- **Всего консультаций** -- количество обработанных обращений.
- **Сгенерировано лидов** -- количество квалифицированных лидов.
- **Среднее время ответа** -- в миллисекундах, среднее по всем ответам агентов.
- **Доля эскалаций** -- процент обращений, переданных живому специалисту.
- **Коэффициент конверсии** -- отношение лидов к общему числу консультаций.

### Дневной тренд

Линейный график отображает количество консультаций и лидов по дням за выбранный период. Позволяет отслеживать динамику нагрузки.

### Топ интентов

Столбчатая диаграмма показывает 5 самых частых намерений пользователей (архитектура, расчет TCO, соответствие 152-ФЗ, миграция, AI Factory).

## Управление разговорами

Страница `/conversations` отображает список всех диалогов с пагинацией.

### Поля таблицы

| Поле | Описание |
|------|----------|
| ID | Уникальный идентификатор |
| Канал | telegram, web, api |
| Статус | active, completed, escalated |
| Интент | Определенное намерение пользователя |
| Дата создания | Время начала диалога |

### Статусы разговоров

- **active** -- диалог в процессе, агент отвечает на вопросы.
- **completed** -- диалог завершен, клиент получил ответы.
- **escalated** -- передан живому специалисту (confidence < 0.6).

## Управление лидами

Страница `/leads` отображает квалифицированных потенциальных клиентов.

### Поля таблицы

| Поле | Описание |
|------|----------|
| Контакт | Имя, email, телефон, компания |
| Квалификация | cold, warm, qualified, hot |
| Интент | Основное намерение клиента |
| Сумма сделки | Оценочная стоимость проекта (руб.) |
| Дата создания | Время создания лида |

### Уровни квалификации

- **cold** -- первичный контакт, намерение не определено.
- **warm** -- проявлен интерес, есть технические вопросы.
- **qualified** -- подтверждена потребность, бюджет обсужден.
- **hot** -- готов к сделке, требуется контакт менеджера.

## Конфигурация агентов

Агенты настраиваются через два механизма:

### 1. Файлы промптов (prompts/*.md)

Каждый агент имеет системный промпт в директории `prompts/`:

| Файл | Агент |
|------|-------|
| `architect.md` | Архитектор облачных решений |
| `cost_calculator.md` | Калькулятор стоимости (TCO) |
| `compliance.md` | Консультант по 152-ФЗ |
| `migration.md` | Специалист по миграции |
| `ai_factory.md` | Консультант AI Factory |
| `human_escalation.md` | Эскалация на живого специалиста |

Для изменения поведения агента отредактируйте соответствующий файл промпта и перезапустите API-контейнер.

### 2. Таблица agent_configs в БД

Для каждого тенанта можно настроить:

- `confidence_threshold` -- порог уверенности (по умолчанию 0.6). Ниже порога -- эскалация.
- `max_turns` -- максимальное количество шагов диалога (по умолчанию 20).
- `rag_collections` -- список RAG-коллекций для поиска.
- `tools` -- список MCP-инструментов, доступных агенту.
- `enabled` -- включен/выключен агент для данного тенанта.

## Резервное копирование

### PostgreSQL

```bash
# Создание дампа
docker compose exec postgres pg_dump -U app cloud_consultant > backup_$(date +%Y%m%d).sql

# Восстановление из дампа
docker compose exec -T postgres psql -U app cloud_consultant < backup_20260301.sql
```

### Qdrant (векторная БД)

```bash
# Создание снимка через API
curl -X POST http://localhost:6333/snapshots

# Список снимков
curl http://localhost:6333/snapshots

# Восстановление из снимка
curl -X PUT http://localhost:6333/snapshots/recover \
  -H "Content-Type: application/json" \
  -d '{"location": "/qdrant/storage/snapshots/<snapshot-name>"}'
```

### MinIO (файловое хранилище)

```bash
# Используйте mc (MinIO Client) для синхронизации
mc alias set local http://localhost:9000 minioadmin minioadmin
mc mirror local/bucket /path/to/backup/
```

### Автоматизация

Рекомендуется настроить cron-задание для ежедневного резервного копирования:

```bash
# /etc/cron.d/cloud-consultant-backup
0 3 * * * root docker compose -f /path/to/docker-compose.yml exec -T postgres pg_dump -U app cloud_consultant | gzip > /backups/db_$(date +\%Y\%m\%d).sql.gz
```

## Решение типичных проблем

### API не отвечает

```bash
# Проверьте статус контейнеров
docker compose ps

# Перезапустите API
docker compose restart api

# Проверьте логи на ошибки
docker compose logs --tail=50 api
```

### Ошибка подключения к БД

```bash
# Проверьте, что PostgreSQL запущен и готов
docker compose exec postgres pg_isready -U app -d cloud_consultant

# Проверьте переменную DATABASE_URL в .env
grep DATABASE_URL .env
```

### Qdrant не отвечает

```bash
# Проверьте состояние
curl http://localhost:6333/healthz

# Перезапустите Qdrant
docker compose restart qdrant
```

### Redis недоступен

```bash
# Проверьте доступность
docker compose exec redis redis-cli ping
# Ожидаемый ответ: PONG

# Проверьте использование памяти
docker compose exec redis redis-cli info memory
```

### Ошибка JWT-токена (401 Unauthorized)

- Убедитесь, что `JWT_SECRET_KEY` установлен и совпадает между перезапусками.
- Проверьте, что токен не истек (время жизни: `JWT_EXPIRE_MINUTES`, по умолчанию 60 мин).
- Очистите кеш браузера и войдите заново.
