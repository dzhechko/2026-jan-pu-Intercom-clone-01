## Сетевая архитектура

### Порты

| Порт | Сервис | Назначение | Внешний доступ |
|------|--------|------------|:--------------:|
| 80 | Nginx | HTTP (редирект на HTTPS) | Да |
| 443 | Nginx | HTTPS | Да |
| 8001 | API | FastAPI (через Docker) | Dev только |
| 3000 | Admin | React-панель | Dev только |
| 5432 | PostgreSQL | База данных | Нет |
| 6333 | Qdrant | REST API | Нет |
| 6334 | Qdrant | gRPC | Нет |
| 6379 | Redis | Кеш | Нет |
| 9000 | MinIO | S3 API | Нет |
| 9001 | MinIO | Веб-консоль | Нет |

### Docker-сети

Проект использует две изолированные Docker-сети:

- **backend** -- внутренняя сеть для API, PostgreSQL, Qdrant, Redis, MinIO. Недоступна извне.
- **frontend** -- сеть для Nginx, API, Admin. Nginx -- единственная точка входа для внешних запросов.

```
Интернет
    |
  Nginx (frontend)
    |
    +-- API (frontend + backend)
    |     |
    |     +-- PostgreSQL (backend)
    |     +-- Qdrant (backend)
    |     +-- Redis (backend)
    |     +-- MinIO (backend)
    |
    +-- Admin (frontend)
```

### Firewall (рекомендации)

```bash
# Разрешить только HTTP/HTTPS
ufw allow 80/tcp
ufw allow 443/tcp
ufw allow 22/tcp   # SSH
ufw default deny incoming
ufw enable
```

В продакшне порты 8001, 3000, 5432, 6333, 6379, 9000 должны быть закрыты для внешнего доступа. Все обращения -- через Nginx.

## Внешние зависимости

| Сервис | Назначение | Обязательный | Примечание |
|--------|------------|:------------:|------------|
| Anthropic API | Основная LLM (Claude) | Да | Требуется API-ключ |
| Telegram Bot API | Канал общения | Нет | Требуется бот-токен |
| Bitrix24 | CRM-интеграция | Нет | Вебхук для синхронизации лидов |
| amoCRM | CRM-интеграция | Нет | Альтернатива Bitrix24 |
| Sentry | Мониторинг ошибок | Нет | Для продакшн-окружения |

### Сетевые вызовы к внешним API

- **Claude API**: `api.anthropic.com` (HTTPS, порт 443).
- **Telegram API**: `api.telegram.org` (HTTPS, порт 443).
- **Bitrix24**: `<домен>.bitrix24.ru` (HTTPS, порт 443).

Убедитесь, что исходящие HTTPS-соединения на порт 443 не блокируются firewall.

## Соответствие 152-ФЗ

### Требования к размещению

Федеральный закон N 152-ФЗ "О персональных данных" требует хранения персональных данных граждан РФ на серверах, расположенных на территории Российской Федерации.

### Реализация в проекте

1. **Размещение сервера**: VPS в московском ДЦ (AdminVPS, HOSTKEY, Selectel или аналоги).
2. **Хранение данных**: PostgreSQL, Qdrant, MinIO -- все на одном сервере в РФ.
3. **Шифрование**: TLS 1.2/1.3 для передачи данных, AES-256-GCM для данных в покое.
4. **Мультитенантность**: изоляция данных по `tenant_id` на уровне всех запросов к БД.
5. **Логирование**: все обращения логируются для аудита, без записи персональных данных в лог.

### Внешние API

Обращения к Claude API (Anthropic) -- это запросы к иностранному сервису. Тексты вопросов пользователей передаются для генерации ответов. Убедитесь, что это соответствует вашей модели обработки ПДн. При необходимости используйте GigaChat как российскую альтернативу (настраивается через `LLM_FALLBACK_MODEL`).
